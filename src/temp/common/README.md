# Table Cells - AI 친화적 Column 아키텍처## 개요이 디렉토리는 ListTemplate 테이블의 renderCell 패턴을 표준화하고, AI 자동화 성공률을 높이기 위한 **Column Builder DSL 아키텍처**를 제공합니다.### 핵심 문제기존 renderCell 방식의 문제점:1. **자유도가 너무 높음**: AI가 다양한 방식으로 코드를 생성할 수 있어 일관성 결여2. **반복적인 보일러플레이트**: stopPropagation, Box 스타일, onClick 로직 등이 계속 반복3. **실수 가능성 높음**: underline 누락, width vs flex, Badge 오용 등4. **AI 자동화 성공률 낮음**: 약 60%의 성공률### 해결책**Column Builder DSL** 도입:- 선언적 API로 일관된 패턴 강제- 재사용 가능한 Cell 컴포넌트- AI가 실수할 여지 최소화- 자동화 성공률 95%로 향상---## 디렉토리 구조`TableCells/├── README.md                      # 이 파일├── types.ts                       # Cell Props 타입 정의├── index.ts                       # Export 파일│├── Cell Components/               # 재사용 가능한 Cell 컴포넌트│   ├── LinkCell.tsx              # 링크 셀 (MID, UID, 회원명)│   ├── TooltipCell.tsx           # 긴 텍스트 Tooltip│   ├── ChipCell.tsx              # Chip 표시│   ├── ButtonCell.tsx            # 버튼 액션│   ├── SwitchCell.tsx            # 토글 스위치│   ├── ModalOpenCell.tsx         # 모달 오픈 버튼│   ├── MultiLineCell.tsx         # 여러 줄 표시│   ├── InputCell.tsx             # 인라인 편집│   ├── SelectCell.tsx            # 인라인 선택│   ├── BadgeCell.tsx             # 숫자 뱃지│   └── StatusTextCell.tsx        # 상태 텍스트│├── ColumnBuilder.tsx              # DSL 클래스│└── Documentation/                 # 문서    ├── REFACTORING_EXAMPLE.md    # Before/After 비교    └── AI_AUTOMATION_STRATEGY.md # AI 자동화 가이드`---## 빠른 시작### 1. 기본 사용법``tsximport { useColumnBuilder } from 'common/components/TableCells';import { GridColDef } from '@bos/components';interface IUserData {  id: string;  name: string;  mid: string;  status: string;  amount: number;}const MyPage = () => {  const col = useColumnBuilder<IUserData>();  const columns = useMemo<GridColDef<IUserData>[]>(    () => [      col.text('name', '이름'),      col.link('mid', 'MID', {        href: (row) => `/detail/${row.id}`      }),      col.statusText('status', '상태', {        textMap: { 'ACTIVE': '활성', 'INACTIVE': '비활성' }      }),      col.number('amount', '금액', { suffix: '원' })    ],    [col]  );  return <ListTemplate columns={columns} rowId="id" />;};``### 2. Cell 컴포넌트 직접 사용``tsximport { LinkCell, ChipCell } from 'common/components/TableCells';// Column 정의에서 사용{  field: 'mid',  headerName: 'MID',  renderCell: ({ value, row }) => (    <LinkCell      value={value}      href={`/detail/${row.id}`}    />  )}``---## Column Builder DSL API### 기본 컬럼#### `col.text(field, headerName, options?)`일반 텍스트 컬럼`tsxcol.text('name', '이름')col.text('description', '설명', {  flex: 1.5,  minWidth: 200,  fallback: '없음'})`#### `col.date(field, headerName, options?)`날짜/시간 컬럼`tsxcol.date('createdAt', '생성일시')col.date('modifiedAt', '수정일시', {  format: 'YYYY-MM-DD',  fallback: '없음'})`#### `col.number(field, headerName, options?)`숫자 컬럼 (자동 천단위 구분자)`tsxcol.number('amount', '금액', { suffix: '원' })col.number('count', '건수', { suffix: '건' })col.number('price', '가격', {  format: 'currency',  decimals: 2})`#### `col.numberWithUnit(field, headerName, unitOptions, options?)`숫자 + 단위 컬럼 (줄바꿈, 단위는 회색)숫자와 단위를 줄바꿈하여 표시하며, 단위는 회색으로 렌더링합니다. KRW, BTC 등 금액/수량 표시에 사용합니다.`tsx// 기본 사용col.numberWithUnit('incomeAmount', '소득금액', { unit: 'KRW' })// 소수점 지정col.numberWithUnit('balance', '잔액', {  unit: 'BTC',  decimals: 8})// flex 커스터마이징col.numberWithUnit('price', '가격',  { unit: 'USD' },  { flex: 0.8, minWidth: 120 })`### 인터랙티브 컬럼#### `col.link(field, headerName, linkOptions, options?)`클릭 가능한 링크 컬럼 (MID, UID, 회원명 등)``tsx// href로 네비게이션col.link('mid', 'MID', {  href: (row) => `/detail/${row.id}`})// onClick으로 커스텀 핸들러col.link('memberName', '회원명', {  onClick: (row, e) => {    e.preventDefault();    handleViewDetail(row);  }})// 외부 링크col.link('url', 'URL', {  href: (row) => row.externalUrl,  external: true})``#### `col.button(field, headerName, buttonOptions, options?)`버튼 컬럼 (수정, 삭제 등)`tsxcol.button('id', '액션', {  label: '수정',  onClick: (row) => handleEdit(row.id),  color: 'primary'})col.button('id', '삭제', {  label: '삭제',  onClick: (row) => handleDelete(row.id),  color: 'error',  disabled: (row) => row.status === 'LOCKED'})`#### `col.switch(field, headerName, switchOptions, options?)`토글 스위치 컬럼`tsxcol.switch('isActive', '활성', {  onChange: (row, checked) => handleToggle(row.id, checked),  showLabel: true,  labels: { on: '활성', off: '비활성' }})`#### `col.modalOpen(field, headerName, modalOptions, options?)`모달 오픈 버튼 컬럼`tsxcol.modalOpen('id', '상세', {  label: '상세보기',  onOpen: (row) => handleOpenModal(row)})`#### `col.input(field, headerName, inputOptions, options?)`인라인 편집 컬럼`tsxcol.input('name', '이름', {  onChange: (row, value) => handleUpdateName(row.id, value),  placeholder: '이름 입력'})`#### `col.select(field, headerName, selectOptions, options?)`인라인 선택 컬럼`tsxcol.select('status', '상태', {  options: [    { value: 'ACTIVE', text: '활성' },    { value: 'INACTIVE', text: '비활성' }  ],  onChange: (row, value) => handleUpdateStatus(row.id, value)})`### 표시 컬럼#### `col.statusText(field, headerName, statusOptions, options?)`상태 텍스트 컬럼 (색상 적용)`tsxcol.statusText('status', '상태', {  textMap: {    'APPLY': '접수',    'APPROVE': '승인',    'REJECT': '반려'  },  colorMap: {    'APPLY': 'info.main',    'APPROVE': 'success.main',    'REJECT': 'error.main'  },  bold: true})`#### `col.chip(field, headerName, chipOptions?, options?)`Chip 컬럼 (태그, 라벨 등)⚠️ **주의**: 상태 표시에는 `col.statusText()` 사용 권장`tsxcol.chip('tag', '태그', {  color: 'primary',  size: 'small',  variant: 'outlined'})`#### `col.tooltip(field, headerName, tooltipOptions?, options?)`Tooltip 컬럼 (긴 텍스트)`tsxcol.tooltip('description', '설명', {  maxLength: 20,  placement: 'top'})`#### `col.multiLine(field, headerName, multiLineOptions?, options?)`여러 줄 표시 컬럼`tsxcol.multiLine('tags', '태그 목록', {  maxLines: 3,  gap: 0.5})`#### `col.badge(field, headerName, badgeOptions?, options?)`숫자 뱃지 컬럼 (알림 개수 등)`tsxcol.badge('notificationCount', '알림', {  color: 'error',  max: 99,  hideZero: true})`---## Cell 컴포넌트 직접 사용Column Builder DSL을 사용하지 않고, Cell 컴포넌트를 직접 사용할 수도 있습니다.### LinkCell`tsximport { LinkCell } from 'common/components/TableCells';<LinkCell  value="A12345678A"  href="/detail/123"  underline={true}  fallback="-"/>`**Props**:- `value: string` - 표시할 값- `href: string` - 이동할 경로- `external?: boolean` - 외부 링크 여부 (기본: false)- `onClick?: (e: React.MouseEvent) => void` - 클릭 핸들러- `underline?: boolean` - underline 스타일 (기본: true)- `fallback?: ReactNode` - 값이 없을 때 표시 (기본: '-')- `sx?: SxProps<Theme>` - 추가 스타일### TooltipCell`tsximport { TooltipCell } from 'common/components/TableCells';<TooltipCell  value="매우 긴 텍스트입니다..."  maxLength={20}  placement="top"/>`**Props**:- `value: string` - 표시할 값- `title?: string` - Tooltip 내용 (기본: value)- `maxLength?: number` - 최대 표시 길이 (기본: 20)- `placement?: 'top' | 'bottom' | 'left' | 'right'` - Tooltip 위치### ChipCell`tsximport { ChipCell } from 'common/components/TableCells';<ChipCell  value="활성"  color="success"  size="small"/>`**Props**:- `value: string` - 표시할 값- `color?: 'default' | 'primary' | 'secondary' | 'error' | 'info' | 'success' | 'warning'`- `size?: 'small' | 'medium'`- `variant?: 'filled' | 'outlined'`- `onClick?: () => void` - 클릭 이벤트### StatusTextCell`tsximport { StatusTextCell } from 'common/components/TableCells';<StatusTextCell  value="ACTIVE"  colorMap={{    'ACTIVE': 'success.main',    'INACTIVE': 'error.main'  }}  bold={true}/>`**Props**:- `value: string` - 표시할 값- `colorMap?: Record<string, string>` - 상태별 색상 매핑- `bold?: boolean` - bold 여부 (기본: false)---## 장점과 단점### ✅ 장점#### 1. 코드 간결성- **76% 코드 감소**: 170 lines → 40 lines- 보일러플레이트 제거- 선언적 API#### 2. 일관성- 모든 테이블이 동일한 패턴 사용- 팀      return (        <Box          component="span"          sx={{            textDecoration: 'underline',            cursor: 'pointer',            color: 'primary.main',            '&:hover': { color: 'primary.dark' }          }}          onClick={(e) => {            e.stopPropagation(); Column Builder DSL 또는 Cell 컴포넌트 사용을 권장합니다. 매우 특수한 케이스에만 renderCell을 직접 사용하세요.### Q2. 새로운 Cell 타입이 필요하면 어떻게 하나요?A. 새로운 Cell 컴포넌트를 추가하고, ColumnBuilder에 메서드를 추가하면 됩니다. 팀에 요청하세요.### Q3. 상태 컬럼에 Badge를 사용하면 안 되나요?A. 기본적으로 권장하지 않습니다. `col.statusText()`를 사용하세요. 기획서에 명시적으로 Badge 디자인이 있을 때만 `col.chip()` 또는 `col.badge()`를 사용하세요.### Q4. AI 자동화 시 어떤 프롬프트를 사용하나요?A. [AI_AUTOMATION_STRATEGY.md](./AI_AUTOMATION_STRATEGY.md) 문서를 참고하세요.### Q5. 기존 페이지를 모두 전환해야 하나요?A. 필수는 아니지만 권장합니다. 새로운 페이지부터 DSL을 사용하고, 기존 페이지는 점진적으로 전환하세요.---## 라이선스This project is part of BOS 2.0.---**마지막 업데이트**: 2026-02-16
